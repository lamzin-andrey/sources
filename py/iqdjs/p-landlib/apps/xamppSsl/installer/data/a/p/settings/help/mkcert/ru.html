Обычно вы видите эту справку в тех случаях, когда FastXAMPPSsl не удалось автоматически добавить SSL сертификат на сайт, расположенный на вашем localhost. Здесь описана установка в Linux Xubuntu 22.04.

Установка

#1 # Установка mkcert
sudo apt install libnss3-tools
curl -s https://api.github.com/repos/FiloSottile/mkcert/releases/latest \
| grep browser_download_url.*linux-amd64 \
| cut -d '"' -f 4 \
| wget -qi -
mv mkcert-v*-linux-amd64 mkcert
chmod +x mkcert
sudo mv mkcert /usr/local/bin/
# Создаем локальный CA
mkcert -install
# Создаем сертификаты для сайтов
#mkcert localhost
#mkcert "localhost" "*.localhost" "site1.local" "site2.local"
#mkcert site1.local
#mkcert site2.local
# И даже так:
mkcert hello.ru
# Сертификаты автоматически доверенные в системе и браузерах

Если в каталоге, в котором вы запустили терминал создались файлы hello.ru.pem и hello.ru-key.pem, значит всё нормально, и FastXamppSsl при последующих созданиях сайтов на localhost сможет добавлять им ssl сертификаты автоматически, прописывая их в /opt/lampp/etc/extra/httpd-vhosts.conf.

У Firefox 146.0 и Google Chrome в Xubuntu 22.04 вообще никаких претензий к такому сертификату не возникает.
