Надо первым делом выполнять `mkcert {SITE_NAME}` чтобы создать локальный сертификат в /tmp.
Если не удалось (например, mkcert не установлен), то выдавать сообщение "mkcert не установлен. Инструкцию по его установке вы можете найти в каталоге приложения в файлеmkcert.install.txt. Сайт добавится без поддержки ssl!". В этом случае логика добавления сайта работает без изменений.

Если же сертификаты сгенерились, придётся усложнить логику.

1 Сертификаты копируются в каталог /opt/lampp/htdocs/{SITE_NAME}/{WWW_DIR_NAME}/ssl		*

2 в v-hosts добавляется такой конфиг (пример):											*
```httpd
<VirtualHost *:443>
	ServerAdmin webmaster@service.moneta.ru
	ServerName service.moneta.ru
	DocumentRoot "/opt/lampp/htdocs/service.moneta.ru/www"
	ScriptAlias /cgi/ /opt/lampp/htdocs/service.moneta.ru/cgi/
	ErrorLog /opt/lampp/htdocs/service.moneta.ru/error.log
	CustomLog /opt/lampp/htdocs/service.moneta.ru/access.log common
	
	SSLEngine on
	SSLCertificateFile "/opt/lampp/htdocs/service.moneta.ru/www/ssl/service.moneta.ru.pem"
	SSLCertificateKeyFile "/opt/lampp/htdocs/service.moneta.ru/www/ssl/service.moneta.ru-key.pem"
</VirtualHost>
```

Очевидно, что в примере  SITE_NAME = 'service.moneta.ru', а WWW_DIR_NAME = 'www', туда надо подставить наши значения из инпутов.

3 Парсим /opt/lampp/etc/php.ini. Находим настройку `openssl.cafile` и получаем её значение. *


```bash
# копируем оригинальный сертификат lampp
		 cp /opt/lampp/share/curl/curl-ca-bundle.crt /etc/ssl/certs/combined-ca-bundle.crt
		 # объединяем с ним серт mkcert
		 cat /home/username/.local/share/mkcert/rootCA.pem | sudo tee -a /etc/ssl/certs/combined-ca-bundle.crt
```
В этом примере `/opt/lampp/share/curl/curl-ca-bundle.crt` - это значение настройки openssl.cafile из php.ini которое мы получили ранее, а username надо заменить на системную $USER. Важно, скрипт p/u.sh будет запускаться из под root (с помощью pkexec), но $USER в скрипте p/rt.sh при этом должен быть текущий.

В p/u.sh пишем запуск `pkexec p/u.sh` (путь лучше вычислить абсолютный) и запускаем p/u.sh

В php.ini производим замену: комментируем оригинальную `openssl.cafile`
и добавляем под ней
openssl.cafile=/etc/ssl/certs/combined-ca-bundle.crt

Это должно обеспечить поддержку ssl при работе на локалхост.
